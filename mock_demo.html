这个演示展示了如何在前端应用中集成Mock日志分析服务。

文档说明

Mock日志分析服务

Mock日志分析服务模拟了大模型分析用户行为日志并返回建议的过程。它从数据库中读取用户最近的行为日志，然后根据预设的模板返回包含建议和命令的响应。

API接口

端点: /api/ai/analyze-logs

方法: POST

请求体:

{
  "userId": "用户ID",
  "pageType": "页面类型(可选)"
}

响应:

{
  "success": true,
  "data": {
    "suggestion": "建议内容",
    "command": "bubble|highlight",
    "confidence": 0.85,
    "mock_mode": true,
    "timestamp": 1234567890,
    "user_id": "用户ID"
  }
}

命令类型

bubble: 显示建议弹窗

highlight: 高亮特定基金(需要fund_id字段)

集成方式

前端可以通过调用analyzeUserLogs函数来使用此服务：

import { analyzeUserLogs } from '../api/ai';

// 在组件中使用
const result = await analyzeUserLogs(userId, pageType);
if (result.command === 'bubble') {
  // 显示建议弹窗
} else if (result.command === 'highlight') {
  // 高亮特定基金
}
